# Project Plan

## W1: Foundation

- Triage all six Leeds sites by junction type, PDF template, DWG availability, and expected complexity.
- Establish the shared `SiteModel` contract.
- Create local conversion workflow from DWG to DXF.
- Draft first client questions and project plan email.

## W2: Single-Site MVP

- Select the simplest Tier 1 site.
- Produce the first end-to-end MAPEM JSON, ASN.1 output, and validation report.
- Create first Full Ground Truth for quality comparison.

## W3: Expansion

- Extend the pipeline to a second site.
- Add batch mode and site configuration.
- Introduce semi-automatic georeferencing and manual correction hooks.

## W4: Generalisation

- Run all six sites through the pipeline.
- Produce two Full Ground Truth examples and four Partial Ground Truth examples.
- Report robustness across site type and data quality.

## W5: Reflection

- Write module-level reflections on what worked, what failed, and what better local authority data would look like.
- Draft final report and demonstration story.

## W6: Delivery

- Package the tool, finalise documentation, prepare demo outputs, and rehearse presentation.
